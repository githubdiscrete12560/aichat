What is PyTorch?

PyTorch is an open-source machine learning library developed by Facebook's AI Research lab (FAIR).

  • Flexible: Dynamic computational graphs for intuitive model building
  • Pythonic: Natural Python syntax and debugging
  • Powerful: GPU acceleration for fast training
  • Research-friendly: Easy prototyping and experimentation
  • Production-ready: Deploy models with TorchScript

PyTorch Architecture

Core Components:

Tensor - Multi-dimensional arrays (like NumPy, but with GPU support)

Autograd - Automatic differentiation for gradient computation

nn.Module - Base class for all neural network layers

Optimizers - Algorithms for parameter updates (SGD, Adam, etc.)

PyTorch uses dynamic computation graphs built on-the-fly during forward pass, enabling flexible architectures.

Why PyTorch for Neural Networks?

  • Automatic Differentiation: Computes gradients automatically via backpropagation
  • GPU Acceleration: Seamlessly move tensors to GPU with .to('cuda')
  • Modular Design: Build networks layer-by-layer with torch.nn
  • Rich Ecosystem: Pre-trained models, datasets, and utilities
  • Dynamic Graphs: Change network architecture during runtime

Perfect for both research and production!

import torch
import torch.nn as nn

class Perceptron(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(2, 1) # 2 inputs ‚Üí 1 output

    def forward(self, x):
        return torch.sigmoid(self.linear(x))

Training the Perceptron

# Create model, loss, and optimizer
model = Perceptron()
criterion = nn.BCELoss() # Binary Cross Entropy
optimizer = torch.optim.SGD(model.parameters(), lr=0.1)

# Training data (XOR problem)
X = torch.tensor([[0.,0.], [0.,1.], [1.,0.], [1.,1.]])
y = torch.tensor([[0.], [1.], [1.], [0.]])

# Training loop
for epoch in range(1000):
    optimizer.zero_grad() # Clear gradients
    output = model(X) # Forward pass
    loss = criterion(output, y)
    loss.backward() # Compute gradients
    optimizer.step() # Update weights

What is a GPU?

GPU (Graphics Processing Unit) is a specialized processor designed for parallel computations.

CPU vs GPU for Neural Networks:

CPU - Few cores (4-16), processes tasks sequentially, good for general computing

GPU - Thousands of cores, processes many tasks simultaneously, excellent for matrix operations

Why GPUs for Deep Learning?

  • Neural networks involve massive matrix multiplications
  • GPUs can compute thousands of operations in parallel
  • Training time: Hours on CPU ‚Üí Minutes on GPU
  • Example: Training a simple network can be 10-100x faster on GPU

import torch
import torch.nn as nn
import torch.optim as optim

# Check device
device = 'cuda' if torch.cuda.is_available() else 'cpu'
print(f'Using device: {device}')

# Define Perceptron
class Perceptron(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(2, 1)
    def forward(self, x):
        return torch.sigmoid(self.linear(x))

# Data (OR gate)
X = torch.tensor([[0.,0.], [0.,1.], [1.,0.], [1.,1.]]).to(device)
y = torch.tensor([[0.], [1.], [1.], [1.]]).to(device)

# Model, loss, optimizer
model = Perceptron().to(device)
criterion = nn.BCELoss()
optimizer = optim.SGD(model.parameters(), lr=0.5)

# Train
for epoch in range(500):
    output = model(X)
    loss = criterion(output, y)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    if (epoch+1) % 100 == 0:
        print(f'Epoch {epoch+1}, Loss: {loss.item():.4f}')

# Test
with torch.no_grad():
    predictions = model(X)
    print('\nPredictions:')
    for i, pred in enumerate(predictions):
        print(f'{X[i].cpu().numpy()} ‚Üí {pred.item():.4f}')

class NeuralNetwork(nn.Module):
    def __init__(self):
        super().__init__()
        self.hidden = nn.Linear(2, 4) # Input ‚Üí Hidden
        self.output = nn.Linear(4, 1) # Hidden ‚Üí Output

    def forward(self, x):
        x = torch.relu(self.hidden(x))
        x = torch.sigmoid(self.output(x))
        return x

Key Takeaways

  • PyTorch provides tensors with GPU support and autograd for automatic differentiation
  • Build networks using nn.Module by defining layers and forward pass
  • Training loop: forward pass ‚Üí compute loss ‚Üí backward pass ‚Üí update weights
  • Hidden layers with activation functions enable learning complex patterns
  • PyTorch's dynamic graphs make debugging and experimentation intuitive
