Introduction to PyTorch

What Is PyTorch?

  • PyTorch is a deep learning framework
  • Designed for:
    • Neural networks
    • Automatic differentiation
    • GPU acceleration
  • Widely used in:
    • Research
    • Education
    • Production systems
Teaching Note: PyTorch focuses on how models learn, not how users interact with them.

PyTorch vs TensorFlow

PyTorch

  • Python-first
  • Dynamic computation graph
  • Easier to debug
  • Preferred for teaching & research

TensorFlow

  • More static
  • Strong production tooling
  • Often used in large-scale deployment
Key Takeaway: PyTorch is ideal for learning and prototyping AI systems.

What Is a Full-Stack AI App?

Components:

  • Frontend: User interface (Flutter)
  • Backend: API + AI logic (FastAPI + PyTorch)
  • Model: Neural network (Perceptron)
  • Storage: Files + Database (SQLite)
Key Idea: Users never interact with PyTorch directly — they interact through an API.

Overall Architecture

Flutter UI
↓ HTTP (REST API)
FastAPI Backend
↓
PyTorch Model
↓
SQLite Database

Responsibilities:

  • Flutter → Input & display
  • FastAPI → Control flow
  • PyTorch → Computation
  • Database → Persistence

Where Does REST API Fit?

REST API acts as the bridge

  • Frontend ↔ Backend communication
  • JSON in / JSON out
  • Stateless requests

Example:

POST /perceptron { "x1": 1, "x2": 2 }

The AI Model: A Perceptron

Model definition:

  • 2 inputs
  • 1 output
  • Sigmoid activation

Math:

y = σ(w₁x₁ + w₂x₂ + b)

Why perceptron?

  • Small
  • Explainable
  • Perfect for teaching fundamentals

File Structure Overview

project/ ├── backend/ │ ├── main.py │ ├── database.py │ ├── perceptron.db │ └── venv/ │ ├── frontend/ │ └── lib/main.dart
Design Principle: Separate UI, logic, and storage.

main.py: Backend Entry Point

Responsibilities:

  • Start FastAPI server

  • Define API endpoints
  • Handle requests

Key libraries:

  • fastapi
  • torch
  • pydantic
  • uvicorn

main.py: Model Code Explained

class Perceptron(nn.Module): def __init__(self): super().__init__() self.linear = nn.Linear(2, 1) def forward(self, x): return torch.sigmoid(self.linear(x))

Explanation:

  • nn.Linear(2,1) → 2-D input
  • forward() → computation logic
  • sigmoid() → output between 0 and 1

main.py: API Endpoint Logic

@app.post("/perceptron") def run_perceptron(data: InputData): x = torch.tensor([[data.x1, data.x2]]) output = model(x)

What happens:

  • 1. Receive JSON
  • 2. Convert to tensor
  • 3. Run model
  • 4. Return result

database.py: Why Do We Need a Database?

Purpose of SQLite:

  • Save inputs
  • Save outputs
  • Enable reproducibility
  • Enable logging & analysis

Why SQLite?

  • No server required
  • One file
  • Perfect for teaching

database.py: Core Concepts

Key ideas:

  • Engine → database connection
  • Session → transaction handler
  • Table → structured storage

Libraries used:

  • sqlalchemy
  • sqlite3 (indirectly)

database.py: Table Definition

class Result(Base): __tablename__ = "results" id = Column(Integer, primary_key=True) x1 = Column(Float) x2 = Column(Float) output = Column(Float)
Teaching Insight: Databases store results, not models.

Where Is Data Stored?

Data Type Storage Location
Model structure main.py
Model weights Memory / .pt file
Inputs & outputs SQLite database
UI state Flutter memory

main.dart: Frontend Responsibilities

What Flutter does:

  • Collect user input
  • Send HTTP request
  • Display response

Flutter does NOT:

  • Run AI
  • Store model
  • Train networks

main.dart: Input Handling

TextField(controller: x1Controller) TextField(controller: x2Controller)
Key Idea:
  • Each input field = one input dimension
  • UI must match model dimension

main.dart: API Call Logic

final response = await http.post( Uri.parse("http://127.0.0.1:8000/perceptron"), body: jsonEncode({...}), );

Flow:

  • 1. Button clicked
  • 2. HTTP request sent
  • 3. JSON parsed
  • 4. UI updated

Displaying Results Safely

if (result != null) ...[ Text("Output: ${result!['output']}") ]

Why ... ?

  • Spread operator
  • Inserts multiple widgets conditionally

Common Backend Errors

Common Frontend Errors

Error Cause Fix

Development Order

Very Important!

  • 1. Activate virtual environment
  • 2. Install backend dependencies
  • 3. Run FastAPI server
  • 4. Open /docs
  • 5. Run Flutter app
  • 6. Test full flow

Educational Takeaways

  • AI ≠ UI

  • REST APIs connect everything
  • Small models are ideal for learning
  • Transparency builds understanding
